Aubit 4GL compiler status

Last updated on 25/06/2005, by Andrej Falout

Aubit compiler is a work in progress, and by the very nature of OpenSource development process, it will remain in such state for foreseeable future. This means that some components may be fully production ready and extensively tested, but some other may be not much then a prototype.

Thanks to the fully modular architecture of Aubit compiler, where practically all functionality is implemented in a form of plug-ins, this varying status levels do not have to be a cause for concern. Additionally, Aubit provides a full automated regression tests suite of over 800 tests, so it easy for user to verify functionality of all components on every Aubit release, in less then 30 minutes.

This page attempts to keep track on progress of various Aubit compiler functionality and components, and quantify there maturity. It will hopefully help you set appropriate level of expectations.

Other symbols used:

Notes:

  • Testing column uses "mostly" for cases where automated testing coverage is extensive, and not "complete", simply because coverage was never explicitly quantified. Keyword "complete" is reserved for functionality where we have defined list of all features, and all tests performed, like we do, for example, for SQL and database features. Therefore, meaning of "mostly" should be interpreted as: As far as we know, most if not all features are tested, but we never explicitly checked that, and there might be some less used features that are not covered by tests.

default

The value that will be used, or the action that will be taken, unless you specify differently. In many SQL and 4GL statements, there is a default action that will be used if you do not specify another; for example, the FETCH statement retrieves the NEXT row unless you specify a different keyword such as PRIOR. Screen forms can specify a default value for input from each field, in case the user fails to enter one.

See also screen form, variable.
